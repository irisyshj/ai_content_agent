## 二、系统设计方案

### 2.1 组织架构

```
📂 采集部（2个Agent）
├── 爬虫专员：极客时间/大家拉/YouTube
└── 资料检索员：Tavily/Perplexity/arXiv

✍️ 创作部（2个Agent）
├── 探索者写手：幽默通俗、高中生能懂
└── 商业分析师写手：麦肯锡风、第一性原理

🎨 制作部（2个Agent）
├── 插画师：DALL-E 3 + Mermaid信息图
└── 视频剪辑师：GPT-SoVITS + Remotion

🔍 质量部（2个Agent）
├── 资深编辑：语言/结构/规范审核（0-10分×3）
└── 第一性原理审核：逻辑/深度/原创审核（0-10分×3）

👔 管理层（1个Agent）
└── 主编：统筹、质量把关、人机协作枢纽

📢 宣发部（1个Agent）
└── 宣发专员：公众号/小红书/飞书/B站/抖音
```

### 2.2 技术架构

```
┌─────────────────────────────────────────────────────────────────┐
│                        表现层                                   │
│  飞书小程序/多维表格    Telegram Bot（可选）                        │
├─────────────────────────────────────────────────────────────────┤
│                        编排层 (n8n)                               │
│  触发器 → 处理器 → 条件判断 → 输出 (HTTP Request Agent)            │
├─────────────────────────────────────────────────────────────────┤
│                        Agent层 (Skills)                            │
│  采集部    创作部    制作部    质量部    管理层    宣发部        │
├─────────────────────────────────────────────────────────────────┤
│                        服务层                                     │
│  飞书API  微信API  Tavily  OpenAI  Whisper  DALL-E  n8n-Webhook     │
├─────────────────────────────────────────────────────────────────┤
│                        数据层                                     │
│  飞书多维表格  飞书云文档  本地文件系统                          │
└─────────────────────────────────────────────────────────────────┘
```

### 2.3 交互流程（3个确认节点）

```
1️⃣ 选题确认
   - 推送3-5个候选选题
   - 你回复编号 / "换一批" / 自定义
   - ✅ 选题确定

2️⃣ 初稿确认（文字）
   - 推送Markdown正文 + 配图计划
   - 你回复"通过" / "修改XX" / "重写"
   - ✅ 文字通过

3️⃣ 配图确认
   - 推送AI生成的配图预览
   - 你回复"全部通过" / "图1重做"
   - ✅ 进入发布流程
```
