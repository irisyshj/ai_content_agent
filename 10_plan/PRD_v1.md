## 1. 产品概述

### 1.1 产品定位

**产品名称**：AI Media Agent System (AI媒体智能体系统)

**产品定位**：面向AI科技科普自媒体的一站式内容生产与分发平台，通过Multi-Agent架构实现从内容采集到多平台分发的全流程自动化。

**核心价值主张**：
> 让内容创作者从重复劳动中解放，专注于洞察与创意；让系统越用越聪明，每次发布都成为下一次的优化基础。

### 1.2 目标用户

| 用户类型 | 描述 | 核心痛点 |
|---------|------|---------|
| **主要用户** | AI科技科普自媒体创作者 | 内容生产耗时、选题困难、多平台运营复杂 |
| **次要用户** | 科技公司内容运营团队 | 需要大量高质量内容、团队协作成本高 |

### 1.3 使用场景

**场景1：日常内容生产**
```
用户每天早上打开系统，查看AI推送的3-5个选题候选
选择一个感兴趣的方向，系统自动开始创作
1小时后收到初稿，审阅后确认
系统自动生成配图、排版，推送到公众号草稿箱
用户稍作修改即可发布
```

**场景2：热点响应**
```
系统监测到行业大热点（如GPT-5发布）
立即推送热点选题
用户确认后，系统快速收集资料、生成深度评论
2小时内完成内容产出，抢占热点窗口
```

**场景3：知识库进化**
```
一篇文章发布后，系统自动采集数据反馈
如果成为爆款（阅读量超均值2倍），自动触发复盘
提取成功要素，更新到知识库
下次创作时，系统优先推荐验证过的方向
```

### 1.4 产品目标

| 阶段 | 目标 | 验收标准 |
|-----|------|---------|
| **MVP (4周)** | 单线跑通 | 极客时间→公众号草稿箱完整链路 |
| **V1.0 (8周)** | 双线+小红书 | 两种风格+小红书+公众号采集 |
| **V2.0 (12周)** | 完整闭环 | 视频+数据回流+知识库进化 |

### 2.2 核心功能详细规格

#### 2.2.1 内容采集模块

| 功能 | 优先级 | 描述 |
|-----|-------|------|
| **P0: 极客时间采集** | 最高 | 从指定课程专栏采集文字、图片、视频转文字 |
| **P0: 大家拉采集** | 最高 | 监控3个指定公众号，采集新文章 |
| **P1: YouTube采集** | 高 | 采集指定频道视频，提取字幕 |
| **P1: 资料增强** | 高 | 基于采集内容进行深度检索补充 |
| **P2: 自定义来源** | 中 | 支持用户自定义文章/视频链接 |

**极客时间采集详细规格**：

| 字段 | 说明 |
|-----|------|
| 输入 | 专栏URL / 课程ID |
| 采集内容 | 文章标题、正文、配图、视频链接 |
| 视频处理 | 下载视频 → Whisper转文字 → 时间戳标注 |
| 输出 | JSON格式结构化数据 |
| 频率 | 按需触发 / 定时增量采集 |
| 反爬策略 | 代理IP池 + 随机延迟 + Cookie管理 |

**大家拉采集详细规格**：

| 字段 | 说明 |
|-----|------|
| 监控数量 | MVP阶段3个公众号 |
| 监控方式 | 每小时检查更新 |
| 采集内容 | 文章正文、作者、发布时间、阅读量、点赞数 |
| 爆款识别 | 阅读>10万 或 点赞>1000 |
| 输出 | JSON格式，同步到飞书素材库 |

#### 2.2.2 内容创作模块

| 功能 | 优先级 | 描述 |
|-----|-------|------|
| **P0: 选题策划** | 最高 | 生成3-5个候选选题，推送用户选择 |
| **P0: 探索者风格创作** | 最高 | 面向技术探索者的通俗科普文 |
| **P1: 商业分析师风格创作** | 高 | 面向管理层的深度商业分析 |
| **P1: 双审核机制** | 高 | 资深编辑+第一性原理双重评分 |
| **P2: 人工协作** | 高 | 选题/初稿/配图三节点人工确认 |

**选题策划详细规格**：

| 步骤 | 说明 |
|-----|------|
| 输入 | 采集内容池 + 用户偏好 |
| 处理 | AI分析素材价值，提取核心观点 |
| 输出 | 3-5个候选选题卡片 |
| 卡片信息 | 标题、来源、推荐风格、核心观点、预计字数 |
| 用户操作 | 选择编号 / "换一批" / 自定义选题 |

**探索者风格创作详细规格**：

| 维度 | 规格 |
|-----|------|
| 目标读者 | 技术探索者，高中生知识水平 |
| 字数范围 | 1500-2000字 |
| 语言风格 | 幽默通俗、生活化比喻、适当Emoji |
| 专业术语 | 每500字不超过3个，必须有解释 |
| 结构要求 | 开篇钩子+核心概念+实际案例+学习建议+互动结尾 |
| 配图要求 | 每800-1000字一张信息图 |
| 金句密度 | 每500-800字一个金句 |

**商业分析师风格创作详细规格**：

| 维度 | 规格 |
|-----|------|
| 目标读者 | 科技公司管理层 |
| 字数范围 | 2000-2500字 |
| 语言风格 | 克制专业、数据驱动、无Emoji |
| 引用要求 | 至少2篇论文/行业报告，APA格式 |
| 结构要求 | 现象引入+第一性原理分析+商业价值+决策建议 |
| 配图要求 | 每600-800字一张专业信息图 |
| 参考文献 | 文末必须附参考文献列表 |

**质量审核详细规格**：

资深编辑审核（语言/结构/规范）：

| 维度 | 权重 | 评分标准 |
|-----|------|---------|
| 语言表达 | 33% | 无语病、表达流畅、无AI味 |
| 结构逻辑 | 33% | 结构清晰、段落过渡自然、逻辑连贯 |
| 规范合规 | 34% | 格式正确、无敏感内容、引用规范 |

第一性原理审核（逻辑/深度/原创）：

| 维度 | 权重 | 评分标准 |
|-----|------|---------|
| 因果链完整 | 33% | 每个结论有推导、逻辑跳跃<2处 |
| 洞察深度 | 34% | 有独特观点、超越表面描述 |
| 原创性 | 33% | 非简单洗稿、有自己分析 |

**评分阈值**：
- 单项≥7分，总分≥24分：通过
- 单项<7分，总分<24分：不通过，返回修改

## 4. 技术选型

## 5. 技术可行性分析

### 5.1 技术风险评估

| 风险 | 概率 | 影响 | 应对策略 |
|-----|------|------|---------|
| **反爬虫对抗** | 高 | 高 | 代理IP池 + 随机延迟 + RPA兜底 |
| **API限流** | 中 | 中 | 请求限流 + 多账号轮换 |
| **AI内容质量不稳定** | 中 | 高 | 双重审核 + 人工确认 |
| **平台API变更** | 中 | 中 | 解耦设计 + 快速适配 |
| **知识库进化复杂** | 低 | 中 | 先做数据记录，后期再做自动化 |

### 5.2 技术难点与解决方案

#### 难点1：极客时间视频转文字

**挑战**：
- 视频可能有加密
- 下载速度慢
- 语音识别准确率

**解决方案**：
```
方案A（推荐）：
1. RPA模拟登录
2. 获取视频播放地址
3. yt-dlp下载
4. Whisper大模型转文字

方案B（兜底）：
1. 人工下载视频
2. 系统调用Whisper API
```

#### 难点2：微信公众号采集

**挑战**：
- 无官方API
- 反爬虫严格
- 内容可能需要登录

**解决方案**：
```
方案A：大家拉（推荐）
- 使用大家拉的服务/数据
- 定期导出到系统

方案B：RPA
- Playwright模拟浏览器
- Cookie管理 + 随机延迟

方案C：RSSHub（备选）
- 部分公众号有RSS
- 不稳定，作为补充
```

#### 难点3：多平台格式适配

**挑战**：
- 各平台格式差异大
- 平台规则频繁变化

**解决方案**：
```
1. 模板化设计
   - 每个平台一个模板
   - 参数化配置

2. 适配器模式
   - PlatformAdapter接口
   - 每个平台一个适配器

3. 快速迭代
   - 用户反馈快速调整
```
