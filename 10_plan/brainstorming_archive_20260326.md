## 一、用户原始需求

### 核心目标
打造一套全自动AI Agent系统，实现从内容采集到多平台分发的完整闭环。

### 具体需求
1. **内容采集**：
   - 极客时间课程专栏（文字、图片、视频→文字稿）
   - 大家拉(dajiala.com)采集公众号文章
   - 自行收集的YouTube内容

2. **内容增强**：检索专业科技网站、论坛、X.com、YouTube等补充资料

3. **双线创作**：
   - 目标读者A：科技公司管理层（第一性原理、商业专业风、引用论文）
   - 目标读者B：技术探索者（亲切幽默、生活化语言、高中生能懂）

4. **多媒体生成**：高质量信息图、配图、AI配音、简单视频

5. **多平台分发**：微信公众号草稿箱、小红书、B站、抖音

### 需要的Agent角色
1. 爬虫专员
2. 资料检索员
3. 主编
4. 插画师
5. 视频剪辑师
6. 宣发专员
7. 资深编辑审核
8. 第一性原理审核
9. 商业内容编辑
10. 科技内容编辑

## 二、参考资料分析

阅读了6个参考文档，了解业内先进实践：

### 2.1 孟健的16个AI员工系统
- **架构**：7个基础Agent + 1个运营总监 + 8个平台Agent
- **核心**：每个平台Agent独立闭环（选题→写稿→平台适配→发布→数据采集→复盘）
- **技术**：Telegram Bot + OpenClaw + VNC远程浏览器
- **成本**：~190美元/月
- **关键**：27个定时任务自动运转，跨Agent通信

### 2.2 林杰的50个Skills系统
- **架构**：12个模块闭环（选题→筛选→框架→素材→写稿→审核→发布→数据→复盘→更新→反哺）
- **核心**：知识库（货）+ Agent（车）+ OpenClaw（高速公路）
- **记忆系统**：SOUL.md、USER.md、MEMORY.md、INVENTORY.md、周工作区
- **两种模式**：自动模式（5分钟/篇）+ 交互模式（练脑子）
- **关键**：系统会自己进化，数据反馈反哺选题

### 2.3 HappyCapy的35个Skills六层架构
- **分层**：采集(8) → 翻译(1) → 创作(9) → 排版(1) → 视觉(6) → 视频(5) → 分发(6)
- **核心创新**：meme-to-article（梗驱动创作）
- **关键**：飞书作为中枢，一鱼多吃

### 2.4 梗驱动创作模式
- **理念**：梗是主线，案例解释梗
- **结构**：梗→解读→案例→洞察
- **三版竞写**：梗解读流 / 反转叙事流 / 连环梗流
- **六维评分**：梗融入度25% + 案例质量15% + 叙事吸引力25% + 洞察深度15% + 结构清晰度10% + 金句密度10%

## 三、用户回答总结

### 3.1 内容来源与优先级

| 来源 | 优先级 | 用途 | 策略 |
|-----|-------|------|------|
| 极客时间课程专栏 | 最高 | 选题灵感 | 作为灵感写原创内容 |
| 大家拉公众号文章 | 次之 | 深度评论 | 素材参考+深度拆解 |
| 自收集公众号/YouTube | 补充 | 素材补充 | 按需使用 |

### 3.2 目标读者定位
- **双线并行**：管理层 + 技术探索者
- **初期偏重**：技术探索者
- **管理层期望**：看完后找作者深入了解 / 发起论坛 / 决策讨论

### 3.3 内容形式优先级

**MVP阶段先跑通**：
1. ✅ 微信公众号长文
2. ✅ 配图/信息图
3. ✅ 小红书图文

**后续阶段**：
- 视频口播脚本
- 完整视频（配音+画面）
- B站/抖音视频

### 3.4 创作方式
- **观点驱动**（不是梗驱动）
- **热点占比**：约10%，很大的热点会参与

### 3.5 人机协作模式
- **初期**：交互模式
- **必须人工确认的节点**：选题确认、初稿确认

### 3.6 大家拉用途
- **A**：监控特定公众号的更新（订阅机制）
- **B**：主动搜索特定主题的文章
- **处理后**：直接作为素材库参考 + 做"深度拆解"（分析为什么爆）

### 3.7 技术栈偏好
- **飞书知识库**
- **Claude Code Skills**
- **工作流工具**（n8n/Make）
- **技术能力**：会编程
- **策略**：先用现成工具快速验证

### 4.1 整体架构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           🎯 一人公司内容系统                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │

│                                                                         │
│   10个Agent分工协作：                                                     │
│   - 采集部：爬虫专员、资料检索员                                        │
│   - 创作部：双线创作（探索者风/管理层风）                               │
│   - 制作部：插画师、视频剪辑师                                          │
│   - 质量部：资深编辑审核、第一性原理审核                                │
│   - 管理层：主编                                                       │
│   - 宣发部：宣发专员                                                   │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4.2 交互流程设计

**节点1：选题确认**
- 每天9点推送3-5个候选选题
- 每个选题包含：来源、推荐风格、核心观点、预期字数
- 用户操作：回复编号 / "换一批" / 自定义

**节点2：初稿确认**
- 推送Markdown正文 + 配图预览 + 小红书版本
- 用户操作："通过" / "修改XX" / "重写"

### 4.3 观点驱动文章结构

```
标题：【观点前置】+ 痛点/悬念

开篇钩子（100-200字）
- 一个场景/数据/矛盾
- 引出核心观点

## 观点阐述
- 核心判断（第一性原理支撑）
- 与主流观点的差异

## 论证A：技术事实
- 专业但通俗的解释

## 论证B：案例支撑
- 实际应用案例

## 论证C：反面思考
- 边界/局限

## 结论+行动建议
```
