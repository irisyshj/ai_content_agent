# AI自媒体超级Agent团队 - 系统架构设计

> 基于 Multi-Agent 架构 + 工作流编排的全自动内容生产系统
> 设计时间：2025年3月

---

## 一、设计理念

### 1.1 核心思想

```
不再使用单体大模型完成所有任务
而是采用"数字员工"协作模式
每个Agent有明确的角色、职责和工作流
```

### 1.2 架构原则

| 原则 | 说明 |
|-----|------|
| **角色拆解** | 每个Agent专注于单一职责 |
| **工作流编排** | 通过引擎串联各数字员工 |
| **RPA+API混合** | 灵活应对不同平台的限制 |
| **Human-in-the-Loop** | 关键节点保留人工审核 |
| **版权合规** | 深度重写而非洗稿 |

### 2.2 Agent角色详解

#### 📂 采集部

| Agent | 代号 | 核心职责 | 技术栈 |
|-------|------|---------|-------|
| **爬虫专员** | `CrawlerAgent` | 极客时间等平台内容采集，视频转文字 | Playwright + Whisper + RPA |
| **资料检索员** | `ResearchAgent` | 全网深度检索，补充专业资料 | Tavily API + Perplexity + arXiv |

**爬虫专员工作流**：
```
1. RPA模拟登录极客时间
2. 定位目标专栏/课程
3. 抓取文字内容 + 图片下载
4. 视频下载（如果有）
5. Whisper语音识别 → 文字稿
6. 输出：结构化内容包
```

**资料检索员工作流**：
```
1. 接收采集内容
2. 提取核心关键词（5-10个）
3. Tavily深度检索（强制site:x.com/arxiv.org）
4. 获取科技大佬最新观点
5. 获取顶会论文摘要
6. 输出：增强资料包
```

#### ✍️ 创作部

| Agent | 代号 | 目标读者 | 写作风格 | 外挂知识库 |
|-------|------|---------|---------|-----------|
| **商业内容编辑** | `BusinessWriterAgent` | 科技公司管理层 | 麦肯锡顾问风，数据驱动，克制专业 | 哈佛商业评论 + 麦肯锡报告 |
| **科技内容编辑** | `TechWriterAgent` | 技术探索者 | 费曼科普风，幽默通俗，生活类比 | 科普作品库 + 比喻库 |

**商业内容编辑Prompt框架**：
```
角色：顶级麦肯锡商业顾问 + AI科学家
任务：使用第一性原理拆解技术，分析商业护城河
风格要求：
- 语言克制、专业、数据驱动
- 使用SWOT/PESTEL等分析框架
- 强制引用学术论文/行业报告
- 文末生成APA格式参考文献
输出：1500-2000字深度分析文
```

**科技内容编辑Prompt框架**：
```
角色：理查德·费曼式科普大师
任务：把复杂AI概念解释给高中生听
风格要求：
- 幽默风趣，亲切如朋友聊天
- 多用生活类比（神经网络→餐厅点菜）
- 适当使用Emoji
- 每段结尾有互动性提问
输出：1200-1500字科普文
```

#### 🎨 制作部

| Agent | 代号 | 核心职责 | 技术栈 |
|-------|------|---------|-------|
| **插画师** | `IllustratorAgent` | 信息图生成，配图创作 | DALL-E 3 + Midjourney + Mermaid.js |
| **视频剪辑师** | `VideoEditorAgent` | 图文视频合成，配音匹配 | Remotion + HeyGen + CapCut API |

**插画师工作流**：
```
1. 接收文字内容
2. 提取核心概念（3-5个视觉点）
3. 转化为图像Prompt
4. 调用DALL-E 3生成配图
5. 生成Mermaid流程图/架构图
6. 输出：图文素材包
```

**视频剪辑师工作流**：
```
1. 接收文字 + 音频 + 图片
2. Remotion组装视频模板
3. 时间轴对齐（音画同步）
4. 添加字幕（自动生成）
5. 渲染输出：1080p/720p视频
```

#### 🔍 质量保证部

| Agent | 代号 | 审核维度 | 评分标准 | 不合格处理 |
|-------|------|---------|---------|-----------|
| **资深编辑审核** | `SeniorEditorAgent` | 语言、结构、规范 | 流畅度/逻辑性/格式 | 返回修改 |
| **第一性原理审核** | `FirstPrincipleAgent` | 逻辑、深度、原创 | 因果链/洞察度/新颖度 | 要求重写 |

**资深编辑审核评分卡**：
```
维度1：语言表达 (0-10分)
- 无语法错误
- 表达流畅自然
- 符合目标读者阅读习惯

维度2：结构逻辑 (0-10分)
- 结构清晰完整
- 段落间逻辑连贯
- 有明确起承转合

维度3：规范合规 (0-10分)
- 引用格式正确
- 无敏感内容
- 符合平台规范

总分 ≥ 24分 通过，否则返回修改
```

**第一性原理审核评分卡**：
```
维度1：因果链完整性 (0-10分)
- 每个结论有明确推导
- 第一性思考清晰
- 无逻辑跳跃

维度2：洞察深度 (0-10分)
- 有独特观点
- 超越表面描述
- 有前瞻性判断

维度3：原创性 (0-10分)
- 非简单洗稿
- 有自己分析
- 融合多源信息

总分 ≥ 24分 通过，否则要求重写
```

#### 👔 主编

| Agent | 代号 | 核心职责 |
|-------|------|---------|
| **主编** | `EditorChiefAgent` | 统筹全局，质量把关，Human-in-the-Loop决策 |

**主编工作流**：
```
1. 接收QA部通过的内容
2. 整体审阅
3. 生成审核报告（摘要+亮点+风险点）
4. 推送人工确认（飞书/微信）
5. 等待人工批准 → 触发宣发
```

#### 📢 宣发部

| Agent | 代号 | 核心职责 | 技术栈 |
|-------|------|---------|-------|
| **宣发专员** | `PromotionAgent` | 多平台格式适配 + 自动发布 | 微信API + RPA + 飞书 |

**宣发专员工作流**：
```
1. 接收主编批准的内容
2. 平台格式适配：
   - 公众号：HTML + CSS
   - 小红书：图片墙 + Emoji
   - B站：视频 + 标题标签
   - 抖音：竖版视频 + 话题
3. 写入飞书/Notion发布队列
4. RPA机器人读取队列
5. 模拟人工操作发布
6. 返回发布结果
```

## 五、Human-in-the-Loop机制

### 5.1 人工介入节点

| 节点 | 介入原因 | 介入方式 | 后续处理 |
|-----|---------|---------|---------|
| **内容确认** | 主编审核后推送 | 飞书/微信点击确认 | 通过→宣发 / 驳回→优化 |
| **异常处理** | 评分不通过 | 人工判断原因 | 调整策略 / 人工修改 |
| **发布确认** | 敏感内容检测 | 人工最终把关 | 确认发布 / 取消 |

### 5.2 人工审核界面设计

```
┌─────────────────────────────────────────────────────────────┐
│  📋 内容审核通知                                            │
```

## 六、版权合规策略

### 6.1 深度重写标准

| 维度 | 要求 | 阈值 |
|-----|------|------|
| **结构重组** | 不是简单同义词替换 | 结构相似度 < 30% |
| **观点原创** | 有独立分析和判断 | 原创观点占比 > 40% |
| **信息融合** | 整合多源信息 | 信息源 ≥ 3个 |
| **价值增量** | 提供新洞察或新视角 | 洞察点 ≥ 2个 |

### 6.2 引用规范

```
原文引用：<10%，必须注明来源
同义改写：允许，但需要重组结构
观点引用：必须标注出处
数据引用：必须标注来源
```

## 七、实施路径

### 7.1 MVP阶段（2-4周）

```
✅ 爬虫专员：极客时间单篇采集
✅ 资料检索员：基础检索
✅ 商业内容编辑：单一风格生成
✅ 资深编辑审核：自动评分
✅ 主编：人工确认机制
✅ 宣发专员：公众号草稿箱
```

### 7.2 完整阶段（8-12周）

```
+ 科技内容编辑：双风格并行
+ 插画师：信息图生成
+ 视频剪辑师：配音+视频
+ 第一性原理审核：深度审核
+ 多平台宣发：小红书/B站/抖音
```
