> 设计时间：2026年3月26日

---

## 一、设计决策汇总

| 决策点 | 选择 | 说明 |
|-------|------|------|
| **D1** | 选题时定风格 | 每篇内容只走一条线，选题阶段决定风格 |
| **D2** | 触发式推送选题 | 灵活推送，不固定时间 |
| **D3** | 初稿先看文字再看图 | 分步确认，降低决策负担 |
| **D4** | 监控3个公众号 | MVP阶段小范围验证 |
| **D5** | 使用n8n | 工作流编排工具 |

## 三、10个Agent角色详细定义

### 3.1 采集部（2个Agent）

#### Agent 1: 爬虫专员 `crawler-specialist`

**职责**：从各源头采集原始内容

| 功能 | 说明 | 技术方案 |
|-----|------|---------|
| 极客时间采集 | 课程专栏文字+图片 | Playwright + RPA |
| 视频转文字 | 视频下载+语音识别 | Whisper API |
| 大家拉采集 | 公众号文章采集 | 大家拉API / 浏览器模拟 |
| YouTube采集 | 视频链接+字幕提取 | yt-dlp + Whisper |
| 内容结构化 | 输出统一JSON格式 | Python脚本 |

**输出格式**：
```json
{
  "source": "geektime",
  "title": "文章标题",
  "content": "正文内容",
  "images": ["url1", "url2"],
  "video": {
    "url": "视频URL",
    "transcript": "文字稿"
  },
  "meta": {
    "author": "作者",
    "date": "2026-03-26",
    "tags": ["标签1", "标签2"]
  }
}
```

#### Agent 2: 资料检索员 `research-specialist`

**职责**：基于采集内容进行深度资料补充

| 功能 | 说明 | 技术方案 |
|-----|------|---------|
| 关键词提取 | 从内容提取5-10个核心关键词 | NLP处理 |
| 深度检索 | Tavily/Perplexity搜索 | API调用 |
| 论文检索 | arXiv相关论文 | arXiv API |
| 科技大佬观点 | X.com/YouTube专家观点 | Social Search API |
| 资料整合 | 输出增强资料包 | 汇总处理 |

**输出格式**：
```json
{
  "original_id": "对应采集内容ID",
  "keywords": ["AI Agent", "第一性原理"],
  "enhanced_materials": {
    "papers": [
      {"title": "论文标题", "url": "链接", "summary": "摘要"}
    ],
    "expert_opinions": [
      {"author": "专家名", "platform": "X/Twitter", "content": "观点", "url": "链接"}
    ],
    "related_articles": [
      {"title": "相关文章", "url": "链接", "relevance": 0.85}
    ]
  }
}
```

### 3.2 创作部（2个Agent）

#### Agent 3: 科技探索者写手 `tech-explorer-writer`

**职责**：为目标读者B（技术探索者）创作内容

**Prompt框架**：
```
角色：你是一位像理查德·费曼一样的科普大师
目标读者：渴望了解AI技术的各行各业技术探索者，高中生知识水平
写作风格：
- 幽默风趣，亲切如朋友聊天
- 多用生活类比（如把神经网络比作餐厅点菜）
- 适当使用Emoji（但不过度）
- 每段结尾可以有互动性提问

文章结构：
1. 开篇钩子（用一个生活场景或矛盾引入）
2. 核心概念（通俗解释，不超过3个专业术语）
3. 实际案例（这个技术怎么用，有什么效果）
4. 学习建议（如果读者想深入了解，可以从哪里开始）
5. 互动结尾（提出一个开放问题）

字数要求：1500-2000字
配图要求：每800-1000字建议一张信息图
```

**输出格式**：
```json
{
  "style": "tech-explorer",
  "article": {
    "title": "标题",
    "content": "Markdown正文",
    "summary": "内容摘要",
    "tags": ["标签"],
    "reading_time": "5分钟"
  },
  "image_prompts": [
    {"position": 800, "description": "配图描述"},
    {"position": 1600, "description": "配图描述"}
  ],
  "xiaohongshu": {
    "title": "小红书标题",
    "content": "小红书文案",
    "tags": ["标签1", "标签2"]
  }
}
```

#### Agent 4: 商业分析师写手 `business-analyst-writer`

**职责**：为目标读者A（科技公司管理层）创作内容

**Prompt框架**：
```
角色：你是一位顶级的麦肯锡商业顾问和AI科学家
目标读者：科技公司管理层，需要深刻理解AI变革
写作风格：
- 克制、专业、数据驱动
- 使用第一性原理分析
- 强制引用学术论文/行业报告
- 文末生成APA格式参考文献

文章结构：
1. 开篇（一个商业现象或矛盾）
2. 第一性原理分析（剥离表象，看本质）
3. 商业护城河分析（技术的商业价值）
4. 行业重塑潜力（对现有格局的冲击）
5. 决策建议（管理层应该关注什么）

字数要求：2000-2500字
引用要求：至少2篇论文/报告
```

**输出格式**：同上，`style` 为 `business-analyst`

### 3.3 制作部（2个Agent）

#### Agent 5: 插画师 `illustrator`

**职责**：生成配图和信息图

| 功能 | 说明 | 技术方案 |
|-----|------|---------|
| 配图生成 | 根据文字内容生成配图 | DALL-E 3 / Midjourney API |
| 信息图生成 | 生成概念图、流程图、对比图 | Mermaid.js + DALL-E |
| 封面生成 | 公众号/小红书封面 | 定制模板 |
| 图片优化 | 统一风格、尺寸调整 | Image Processing |

**信息图类型**：
- 对比图：两个概念并置对比
- 流程图：技术原理/工作流程
- 概念图：抽象概念可视化
- 数据图：数据/趋势展示
- 边界图：技术的适用边界

#### Agent 6: 视频剪辑师 `video-editor`

**职责**：视频内容制作（第二阶段）

| 功能 | 说明 | 技术方案 |
|-----|------|---------|
| 视频脚本生成 | 文章转视频脚本 | GPT-4 |
| AI配音 | 文字转语音 | GPT-SoVITS / MiniMax |
| 视频合成 | 脚本+配音+画面→视频 | Remotion / HeyGen |
| 字幕生成 | 自动生成字幕 | Whisper |

### 3.5 管理层（1个Agent）

#### Agent 9: 主编 `editor-in-chief`

**职责**：统筹全局，质量把关，人机协作枢纽

**功能**：
1. 接收通过审核的内容
2. 整体审阅

4. 推送人工确认
5. 等待批准后触发宣发

**推送报告格式**：
```json

  "timestamp": "2026-03-26T10:30:00Z",
  "article": {

    "word_count": 1800,

  },
  "reviews": {

  "highlights": [
    "观点1：AI Agent的核心价值...",
```

### 4.2 n8n工作流设计

#### 工作流1：内容采集与选题

```
触发：定时任务（每2小时） OR 手动触发

节点：
1. HTTP Request → 调用爬虫Agent
2. Function → 处理采集结果
3. HTTP Request → 调用资料检索Agent
4. Function → 生成选题候选
5. Send to Feishu → 推送选题供选择
```

#### 工作流2：内容创作与审核

```
触发：飞书选题确认

节点：
1. Get from Feishu → 获取选题
2. Switch → 判断风格
3. HTTP Request → 调用对应写手Agent
4. HTTP Request → 资深编辑审核
5. HTTP Request → 第一性原理审核
6. If → 判断是否通过
7. Send to Feishu → 推送初稿确认
```

#### 工作流3：视觉制作与发布

```
触发：初稿通过

节点：
1. HTTP Request → 调用插画师Agent（生成配图）
2. Send to Feishu → 配图确认
3. HTTP Request → 公众号排版
4. HTTP Request → 小红书适配
5. Function → 保存到飞书
6. HTTP Request → 推送到公众号草稿箱
```

```
- id: 记录ID

- views: 阅读量
- likes: 点赞数
```
